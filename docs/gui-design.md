# GUI 设计

## 定位

GUI 是中文管理控制台，用于观察、诊断和发出管理命令。GUI 不直接拼包、不直接修改数据库、不直接操作 socket。所有 GM 功能必须通过 Application Command 执行。服务端核心必须能够无 GUI 部署运行。

## 模块

### 服务总览

- 显示服务启动状态、监听端口、在线连接数、在线角色数、资源加载状态、数据库状态、最近错误。
- 所有状态来自 Host/Application/Diagnostics 的只读视图。

### 发包/接包日志

字段必须包含：

| 字段 | 说明 |
| --- | --- |
| 时间 | 日志记录时间 |
| 方向 C2S/S2C | 客户端到服务端或服务端到客户端 |
| 连接 ID | 网络连接标识 |
| 账号 | 已识别账号，未知时为空 |
| 角色 | 已选角色，未知时为空 |
| Session 状态 | 当前会话状态 |
| Raw Hex | 原始包十六进制 |
| Decoded Hex | 解码后十六进制 |
| Header | 包头信息 |
| Length | 长度字段 |
| AC | 主命令号 |
| SubAC | 子命令号 |
| 协议名称 | 已知协议名称，未知时标记 unknown |
| 行为说明 | 处理器给出的中文行为说明 |
| 字段解析 | 结构化字段列表 |
| Handler | 处理器名称 |
| 状态变化 | Session 或业务状态变化 |
| 资源校验结果 | 相关资源映射或校验结果 |
| 来源标记 | `trace:client`、`reference:muayad`、`reference:wlophoenix`、`assumption`、`unknown` |
| 处理结果 | 成功、拒绝、忽略、异常、待验证 |

### 协议解码

- 展示 AC/SubAC、字段解析、来源标记、状态、风险。
- 未经 trace 验证的行为显示为 pending-target-client-trace。

### 行为说明

- 用中文解释每个包的业务含义、触发条件、状态前后变化。
- 不把参考服务端行为显示为 confirmed，除非存在真实客户端 trace。

### Session 监控

- 显示连接 ID、账号、角色、当前状态、登录阶段、最近收发包、断线原因。

### 在线玩家

- 显示账号、角色、地图、坐标、状态、延迟、最后活动时间。
- 管理动作通过 GM 面板发出 Application Command。

### 资源查看器

- 查看 Item、Skill、Npc、Scene、Eve、Ground、Talk、Compound、Formula 的加载摘要和校验结果。
- 不嵌入或复制客户端资源文件。

### 地图/NPC/任务查看器

- 查看地图、NPC、任务数据的解析结果、来源、校验状态。
- 与真实 trace 不一致时标记风险。

### GM 面板

- 提供踢下线、广播、定位、状态查询等管理命令入口。
- 所有操作都必须转成 Application Command，记录审计日志。

### 服务配置

- 管理监听地址、日志级别、资源路径、数据库连接、Replay 输出位置。
- 敏感信息不得明文展示或提交。

### 日志导出

- 导出结构化服务日志、错误日志、包日志摘要。
- 导出前必须支持脱敏。

### Replay 导出

- 只导出 sanitized replay。
- 导出内容必须适合提交到仓库或明确标记为本地私有文件。
